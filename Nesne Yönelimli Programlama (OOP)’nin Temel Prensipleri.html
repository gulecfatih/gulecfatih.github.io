NESNE YÖNELİMLİ PROGRAMLAMA (OOP)’NIN TEMEL PRENSİPLERİ

Sınıf ve Nesne İlişkisi

  1. INHERITANCE (KALITIM/MİRAS ALMA)

    Kalıtım, bir sınıfın başka bir sınıftan özelliklerini ve metotlarını miras alabilmesini sağlar. Bu, kodun tekrar kullanılmasını artırır ve sınıflar arasında hiyerarşik ilişkiler kurulmasına olanak tanır.

    // Temel sınıf (ana sınıf)
public class LivingBeing
{
    public void Reproductive()
    {
        Console.WriteLine("Increase In Numbers");
    }
}

// Türetilmiş sınıf (alt sınıf)
public class Human : LivingBeing
{
    public void Think()
    {
        Console.WriteLine("I think therefore I am.");
    }
}

class Program
{
    static void Main()
    {
        // Human sınıfından bir nesne oluşturuyoruz.
        Human insan = new Human();

        // LivingBeing sınıfındaki metodu çağırabiliriz.
        insan.Breathe();

        // Human sınıfındaki özel metodu çağırabiliriz.
        insan.Think();
    }
}


  2. ENCAPSULATION (KAPSÜLLEME/SARMALAMA)

    Kapsülleme, sınıfların içerdikleri veriyi ve bu veriye erişim metotlarını bir araya getirme prensibidir. Bu, erişim kontrolü sağlayarak ve güvenlik önlemleri ekleyerek veriye doğrudan erişime karşı koruma sağlar.


    Bu özelliği sağlayan belirleyiciler (access modifiers)

    • Public: Her yerden erişilebilir.
    • Private: Sadece tanımlandığı sınıf içerisinden erişilebilir.
    • Internal: Sadece bulunduğu projede erişilebilir.
    • Protected: Sadece tanımlandığı sınıfta ya da o sınıfı miras alan sınıflardan erişilebilir.
    • Protected Internal: Sadece tanımlandığı sınıfta ya da o sınıfı miras alan sınıflardan erişilebilir. Ayrıca tanımlamanın aynı proje içerisinde olma şartı yoktur. Protected’dan farkı budur.
    • Eğer erişim belirleyici belirtilmemişse, sınıflar internal’dır.

    using System;

public class Car
{
    
    // Araba sınıfının özellikleri
    public string Brand { get; set; }
    public string Model { get; set; }
    public int ProductionYear { get; set; }

  
    // Araç bilgilerini görüntüleme yöntemi
    public void DisplayCarInfo()
    {
        Console.WriteLine($"Car Information: Brand - {Brand}, Model - {Model}, Production Year - {ProductionYear}");
    }
}

class Program
{
    static void Main()
    {
        // Araba sınıfına ait bir nesne oluşturuluyor
        Car car = new Car();

        
        // Araç özelliklerine değer atama
        car.Brand = "Tofaş";
        car.Model = "Şahin";
        car.ProductionYear = 1996;

        // Araç bilgileri görüntüleniyor
        car.DisplayCarInfo();
    }
}

  3. ABSTRACTION (SOYUTLAMA)

    Soyutlama, karmaşık sistemlerin temel özelliklerine odaklanarak gereksiz detayları gizler. Soyut sınıflar veya arabirimler kullanarak, sadece önemli detayları ortaya çıkarır ve kullanıcılara basitleştirilmiş bir görünüm sunar.


    public abstract class Shape
{
    // Soyut method
    public abstract double CalculateArea();
}

public class Rectangle : Shape
{
    public double Length { get; set; }
    public double Width { get; set; }

    public override double CalculateArea()
    {
        return Length * Width;
    }
}

  4. POLYMORPHİSM (ÇOK BİÇİMCİLİK)

    Çok biçimlilik, aynı adı taşıyan fakat farklı davranışlara sahip metotların kullanılabilmesini ifade eder. Bu prensip, aynı metot adını kullanarak farklı türde nesnelerle çalışmayı mümkün kılar ve kodun daha esnek olmasını sağlar.


    using System;

public class Shape
{
    public virtual void Draw()
    {
        Console.WriteLine("Drawing a shape.");
    }
}

public class Circle : Shape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a circle.");
    }
}

public class Square : Shape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a square.");
    }
}

    OOP hakkında temel bilgileri edindik, şimdi bu bilgileri kullanarak bir evrim simülasyonu oluşturmak için bir projeye başlayalım.

    Projemizin ilk aşamasında, temel iskelet yapımızı oluşturmak adına kullanacağımız fonksiyonları belirlemek amacıyla dört adet interface tanımlıyoruz. Bu arayüzler, projemizin ana yapı taşlarını temsil edecek ve kendi içerisinde belirli görevleri ifade edecektir.


    interface IAnimal
{
    void DisplayInfo(); // Hayvanın bilgilerini ekrana yazdıran metot
    void GetSpecialProperty();
}

    
interface IEvolution
{
    void GetSpecialProperty(); // Hayvanın evrimleşme özelliklerini döndüren metot
}

    
interface IHomoSapiens : IAnimal, IEvolution
{
    void Think(); // İnsanın düşünme yeteneğini simüle eden metot
}

    
interface IRattus : IAnimal
{
    void Climb(); // Hayvanın tırmanma davranışını simüle eden metot
}

    Şimdi, bu interfacelerde tanımlanan soyut kavramları default bir değerde kullanabilmek için, Mammal sınıfımızı oluşturuyoruz. Mammal sınıfı, projemizin temel soyutlamalarını ve evrimsel süreçte ortak olan özellikleri içerir. Bu sınıf, IAnimal ve IEvolution interfacelerini implemente ederek, genel bir temsil sağlar ve projemizin ana soyut yapılarını belirler.


    // Memelilerin temel sınıfı olan Mammal
abstract class Mammal : IAnimal, IEvolution
{
    protected string _name; // Hayvanın adı

    protected Mammal(string name)
    {
        _name = name;
    }

    public virtual void DisplayInfo()
    {
        Console.WriteLine($"{_name} bir memeli hayvandır.");
    }
    public virtual void GetSpecialProperty()
    {
        Console.WriteLine($"Özelliği Yok");
    }
}

    Şimdi Homosapiens sınıfımızı tanımlıyoruz ve bu aşamada Mammal sınıfını ve IHomosapiens interfacesini implemente ederek yapımızı kuruyoruz. Homosapiens sınıfı, insan türünü temsil eder ve evrimsel süreçte Mammal sınıfının genel özelliklerini taşır. IHomosapiens arayüzünü implemente ederek, insan türüne özgü davranışları ve özellikleri tanımlarız. Bu adımla, projemizin evrimsel aşamalarını temsil eden Homosapiens sınıfını başarılı bir şekilde oluşturmuş oluruz.


    class HomoSapiens : Mammal, IHomoSapiens
{
    private int _iq; // Homo Sapiens'in IQ değeri

    // Yapılandırıcı metot
    public HomoSapiens(string name, int iq) : base(name)
    {
        _name = name;
        _iq = iq;
    }

    // IAnimal arabiriminden uygulanan metotlar
    public override void DisplayInfo()
    {
        Console.WriteLine($"{_name} türünden bir canlıdır.");
    }

    // IEvolution arabiriminden uygulanan metot
    public override void GetSpecialProperty()
    {
        Console.WriteLine($"Yüksek IQ: {_iq}");
    }

    public void UseTools()
    {
        Console.WriteLine($"{_name} araç kullanıyor.");
}
}

    Şimdi Rattus sınıfımızı tanımlıyoruz ve bu aşamada Mammal sınıfını ve IRattus interfacesini implemente ederek yapımızı kuruyoruz. Rattus sınıfı, bir kemirgen türünü temsil eder ve evrimsel süreçte Mammal sınıfının genel özelliklerini bünyesinde barındırır. IRattus arayüzünü implemente ederek, kemirgen türüne özgü davranışları ve özellikleri tanımlarız. Bu adımla, projemizin evrimsel aşamalarını temsil eden Rattus sınıfını başarılı bir şekilde oluşturmuş oluruz.


    class Rattus : Mammal, IRattus
{
    private bool _canGnaw; // Rattus'un kemirme yeteneği

    // Yapılandırıcı metot
    public Rattus(string name, bool canGnaw) : base(name)
    {
        _name = name;
        _canGnaw = canGnaw;
    }

    public void Climb()
    {
        Console.WriteLine($"{_name} tırmanıyor.");
    }

    // IAnimal arabiriminden uygulanan metotlar
    public override void DisplayInfo()
    {
        Console.WriteLine($"{_name}'tur.");
    }

    // IEvolution arabiriminden uygulanan metot
    public override void GetSpecialProperty()
    {
        Console.WriteLine($"Kemirme Yeteneği: {_canGnaw}");
    }
}

    Son olarak, projemizin yapısını uygun bir biçimde tamamlayabilmek için Homosapiens ve Rattus sınıflarını main fonksiyonunda çağırarak sürecin işlemesini sağlıyoruz. Bu adımda, oluşturduğumuz sınıfların ve arayüzlerin işleyişini kontrol etmek, projemizin evrimsel sürecini anlamak ve doğru bir şekilde işlediğinden emin olmak amacıyla main fonksiyonunu kullanıyoruz. Bu sayede, projemizin tamamlayıcı aşamalarını bir araya getiriyor ve oluşturduğumuz evrimsel yapının başarıyla çalıştığını doğruluyoruz.


    static void Main(string[] args)
{
    // HomoSapiens sınıfından bir nesne oluşturulur ve ilgili metotlar çağrılır.
    HomoSapiens homoSapiens = new HomoSapiens("Homo Sapiens", 120);
    homoSapiens.DisplayInfo(); // Polymorphism özelliği kullanıldı
    homoSapiens.UseTools(); // HomoSapiens sınıfına özgü davranış
    homoSapiens.GetSpecialProperty();

    Console.WriteLine();

    // Rattus sınıfından bir nesne oluşturulur ve ilgili metotlar çağrılır.
    Rattus rattus = new Rattus("Rattus", true);
    rattus.DisplayInfo(); // Polymorphism özelliği kullanıldı
    rattus.Climb(); // Mammal sınıfından gelen davranış
    rattus.GetSpecialProperty();

    Console.ReadLine();
}

    Bu adımlarla, Nesne Yönelimli Programlama (OOP) mantığına uygun olarak evrimsel sürecin küçük bir yapısını kurgulamış ve uygulamış oluyoruz. Sınıflar, arayüzler ve kalıtım gibi OOP prensiplerini kullanarak, projenizdeki soyut kavramları ve yapıları birbirine bağlı bir şekilde organize etmiş oluyorsunuz. Bu, kodunuzun düzenli, esnek ve sürdürülebilir olmasına katkıda bulunur.
